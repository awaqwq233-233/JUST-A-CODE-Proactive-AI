与最终目标的差距 (The Gap)
如果要将 J.A.C. (Just A Code) 打造为真正的"全功能语音助手"（类似 Jarvis 或 Google Assistant 的本地版），目前还存在以下显著差距：

1. 交互方式依然“被动” :
   - 现状 : 需要按空格键触发。
   - 目标 : 唤醒词 (Wake Word) + VAD (语音活动检测) 。实现“嘿 J.A.C.”直接唤醒，并能自动判断你什么时候说完话，无需物理按键。
2. 缺乏“手” (Function Calling) :
   - 现状 : 只能“看”和“说”。
   - 目标 : 工具调用能力 。例如：查询实时天气、控制电脑音量、打开网页、管理日程、搜索文件等。目前 LLM 仅输出文本，没有执行系统命令的能力。
3. 记忆力有限 (Memory) :
   - 现状 : 只有短期对话历史。重启程序后，它不记得你是谁，也不记得之前的对话。
   - 目标 : 长期记忆 (Vector DB / JSON) 。记住用户的喜好、之前的对话摘要，实现个性化陪伴。
4. 响应速度 (Latency) :
   - 现状 : 录音(4s) -> 识别(1-2s) -> 思考(2-5s) -> 合成(2-4s)。一轮对话可能需要 10秒+。
   - 目标 : 流式对话 (Streaming)。一边思考一边生成语音，大幅降低感知延迟。
