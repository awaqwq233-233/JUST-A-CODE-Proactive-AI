# 记忆功能（用户指南）

> 这是 J.A.C. 记忆功能的用户向说明。运维 / 排障见 `runbook.md`，隐私细则见 `privacy.md`，数据格式见 `schema.md`。

## 记忆是什么

J.A.C. 会**在本地**记住一些关于你的长期信息，像一本只存在你电脑上的小抄——它不联网、不上云，你随时能看、能导出、能清空。

## 助手记住了什么

记忆分 5 类（用户能懂的语言）：
- **用户画像（profile）**：你是谁，比如职业、背景。
- **偏好（preference）**：你喜欢怎样被对待，比如用中文回复。
- **约定（convention）**：我们之间的习惯，比如每周同步进度。
- **事件（event）**：发生过的事，比如某次演示。
- **主题（topic）**：你持续关心的方向。

两种来源要分清：
- **你主动说的（explicit）**：你明确告诉它的。
- **它推断的（inferred）**：它从对话里猜出来的——这种你可以单独一键清空，保留你主动说的。

## 如何知道被记住

- 应用内会提示"已记住 X"；你也能随时在「查看」里看到全部记忆。

## 如何查看 / 导出 / 清除

- **查看**：应用内「查看」，或直接打开本地文件（路径见 `runbook.md` §3.3）。
- **导出**：应用内「导出」生成备份；迁移新机器用（见 `runbook.md` §5 / §6）。
- **清除**：应用内可清除某条 / 按来源 / 全部。**清除不可逆**（见 `privacy.md` §4.5）；清全部前建议先导出。

## 隐私说明（简短）

- 记忆只存在你本地，默认不上传到任何云服务。
- 你能完全控制：看得到、导得出、清得掉。
- 涉及人物 / 转录的敏感内容会被标记，你可单独清理系统推断的部分。
- 更细的存储位置、加密、日志控制见 `privacy.md`。

## 常见问题

- **记忆会自动发给云端吗？** 不会。v1 明文存本地、不出网。
- **能完全关掉吗？** 能。关闭记录在 `consent.json`，关闭后不再新写，已记内容仍可手动清除。
- **误记了怎么办？** 应用内清除那一条，或按来源批量清（如清掉所有"推断"），见 `runbook.md` §4。
- **换电脑怎么办？** 导出 / 复制整个记忆目录到新机器，见 `runbook.md` §6。
